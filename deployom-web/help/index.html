Server

Update Environment

Used for update environment name and title shown in the interface.

Sites

The "Sites" tab allow to add and remove sites.
Any number of Sites are supported, but from performance point of view and network topology should be configured as local and remote.
After adding or removing Sites application should be restarted to schedule Custom Jobs execution.

Add Local Site

For Site with hosts in the same network as host where application installed.

Add Remote Site

For Site with hosts in different network as host where application installed.

Users

The "Users" tab allow to add and remove Users.

Modules

The "Modules" tab allow to register and remove modules.

Logging

The "Logging" tab displays critical web application messages (host unavailability, major release errors, etc).
Tab for each Site contain Service Flows, Modules and Events.

Configured Services

The "Configured Services" tab displays Hosts with online and offline Services for one site. Services can be changed in the "Change Layout" tab.
Each service is a button with left click menu. All services have "Online" menu which shows information about online status of the Service.
Besides "Online" Service specific actions implemented.

Change Layout

The "Change Layout" tab displays all services for each Host Type according to Platform Release.
Each Host is a button with left click menu "Update Host" and "Remove Host".
Each Service is a button to manually add or remove.

Update host

Host type and IP can be updated using left click menu "Update Host".

Remove host

Host can be removed using left click menu "Remove Host". Host definition will be removed permanently and can not be recovered.
Tab for each Release will contain Release definitions and Services.

Custom Jobs

All jobs have "period" (how often job will be run) and "start" (when job will be run after application start) parameters.
Each Job schedule in separate thread.

Service Connections

Connections between Services.

Service Flows

Based on requirement any Service Flow can be configured.

Site Map

Full Site configuration with Service mapping.
